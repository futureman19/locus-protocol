# REM-018: Restrict Tribal Council Proposal Authorization

**Fixes:** FINDING-018
**Priority:** LOW
**Effort:** Small (< 1 hour)

## Current Code

```elixir
def can_propose?(%City{} = city, pubkey, :tribal_council) do
  pubkey == city.founder_pubkey or pubkey in city.citizens
end
```

## Recommended Fix

Add a `council_members` field to the City schema and check against it:

```elixir
def can_propose?(%City{} = city, pubkey, :tribal_council) do
  pubkey == city.founder_pubkey or pubkey in (city.council_members || [])
end
```

This requires:
1. Adding `council_members: [binary()]` to the City schema
2. Implementing council election logic (founder selects 2 council members)
3. Storing council member pubkeys in the city state

The spec says "founder + 2 elected" — the election mechanism needs to be defined. As a minimum viable fix, the founder could appoint council members directly until elections are implemented.
