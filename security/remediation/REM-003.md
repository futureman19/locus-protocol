## Option A: Two-output enforcement via covenant (complex)

BSV supports OP_PUSH_TX-style covenants that can enforce output amounts and destinations. This would require:
1. The script to introspect the spending transaction
2. Verify that output[0] sends >= 10% to the protocol treasury address
3. Verify that output[1] sends <= 90% to the owner

This is technically possible on BSV but adds significant script complexity.

## Option B: Require protocol co-signature for emergency path (recommended)

```elixir
def build_lock_script(owner_pubkey, lock_height, protocol_treasury_pubkey) do
  redeem_script = Script.new()
  |> Script.push_op(:OP_IF)
  # Normal unlock path (owner only, after CLTV)
  |> Script.push_int(lock_height)
  |> Script.push_op(:OP_CHECKLOCKTIMEVERIFY)
  |> Script.push_op(:OP_DROP)
  |> Script.push_data(Base.decode16!(owner_pubkey, case: :mixed))
  |> Script.push_op(:OP_CHECKSIG)
  |> Script.push_op(:OP_ELSE)
  # Emergency path: requires BOTH owner AND protocol signatures
  # Protocol only co-signs if 10% penalty output is present
  |> Script.push_int(10)
  |> Script.push_op(:OP_CHECKSEQUENCEVERIFY)
  |> Script.push_op(:OP_DROP)
  |> Script.push_int(2)  # 2-of-2 multisig
  |> Script.push_data(Base.decode16!(owner_pubkey, case: :mixed))
  |> Script.push_data(Base.decode16!(protocol_treasury_pubkey, case: :mixed))
  |> Script.push_int(2)
  |> Script.push_op(:OP_CHECKMULTISIG)
  |> Script.push_op(:OP_ENDIF)

  Script.p2sh(redeem_script)
end
```

The protocol's signing service only co-signs emergency unlocks where the transaction includes the correct 10% penalty output. This moves enforcement from pure script to a semi-trusted model, but the protocol cannot steal funds (it can only block emergency unlocks, not the normal path).
