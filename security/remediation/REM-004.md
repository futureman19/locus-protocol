# REM-004: Reconcile Node and Core Staking Scripts

**Fixes:** FINDING-004
**Priority:** HIGH
**Effort:** Medium (2-4 hours)

## Problem

`node/lib/locus/staking.ex` builds a simple CLTV script (no emergency path), while `core/lib/locus/staking.ex` builds a complex OP_IF/OP_ELSE script. They are incompatible.

## Recommended Fix

The node module should import and use the core module's staking functions rather than reimplementing them.

### Step 1: Add core as a dependency

```elixir
# node/mix.exs
defp deps do
  [
    {:locus_core, path: "../core"},
    # ...
  ]
end
```

### Step 2: Delegate to core

```elixir
# node/lib/locus/staking.ex
defmodule Locus.Staking do
  # Delegate to core for script building
  defdelegate build_lock_script(owner_pubkey, lock_height, treasury_address),
    to: Locus.Core.Staking

  # Node-specific slashing logic stays here
  def calculate_slash(stake_amount, condition) do
    # ...
  end
end
```

### Step 3: Update tx_builder.ex

```elixir
# node/lib/locus/tx_builder.ex:74-75
# Change from:
redeem_script = Staking.build_lock_script(lock_height, owner_pubkey_binary)
# To:
redeem_script = Locus.Core.Staking.build_lock_script(
  Base.encode16(owner_pubkey_binary, case: :lower),
  lock_height,
  protocol_treasury_address
)
```

### Step 4: Test compatibility

Write a test that:
1. Builds a lock script using core
2. Builds a lock script using node
3. Asserts they produce identical P2SH addresses
