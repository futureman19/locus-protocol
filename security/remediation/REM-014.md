# REM-014: Use Integer Arithmetic for Vote Tallying

**Fixes:** FINDING-014
**Priority:** MEDIUM
**Effort:** Small (< 1 hour)

## Current Code

```elixir
# core/lib/locus/governance.ex:167
proposal.votes_for / (proposal.votes_for + proposal.votes_against) >= threshold
```

## Recommended Fix

Use cross-multiplication to avoid float division:

```elixir
def tally(%Proposal{} = proposal, %City{} = city) do
  {threshold_num, threshold_den} = threshold_fraction(proposal.proposal_type)
  {quorum_num, quorum_den} = quorum_fraction(city.phase)

  total_votes = proposal.votes_for + proposal.votes_against + proposal.votes_abstain
  quorum_needed = div(city.citizen_count * quorum_num + quorum_den - 1, quorum_den)

  active_votes = proposal.votes_for + proposal.votes_against

  cond do
    total_votes < quorum_needed ->
      :pending
    active_votes == 0 ->
      :pending
    # Cross-multiply to avoid float: votes_for / active >= num / den
    # becomes: votes_for * den >= num * active
    proposal.votes_for * threshold_den >= threshold_num * active_votes ->
      :passed
    true ->
      :rejected
  end
end

# Return thresholds as integer fractions
defp threshold_fraction(:parameter_change), do: {51, 100}
defp threshold_fraction(:contract_upgrade), do: {66, 100}
defp threshold_fraction(:treasury_spend), do: {51, 100}
defp threshold_fraction(:constitutional), do: {75, 100}
defp threshold_fraction(:emergency), do: {7, 12}
defp threshold_fraction(_), do: {51, 100}

defp quorum_fraction(:town), do: {60, 100}
defp quorum_fraction(:city), do: {40, 100}
defp quorum_fraction(:metropolis), do: {51, 100}
defp quorum_fraction(:village), do: {67, 100}
defp quorum_fraction(_), do: {1, 1}
```

This eliminates all floating-point operations from the consensus-critical governance path.
