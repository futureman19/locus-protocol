## Recommended Fix

Sign the OP_RETURN payload with the founder's private key before encoding:

```typescript
import { PrivateKey, Hash } from '@bsv/sdk';

static buildCityFound(params: CityFoundParams, privateKey: PrivateKey): Buffer {
  // Build the payload without signature first
  const payload = {
    name: params.name,
    description: params.description || '',
    location: { lat: params.lat, lng: params.lng, h3_res7: params.h3Res7 },
    founder_pubkey: params.founderPubkey,
    policies: params.policies || {},
  };

  // Sign the serialized payload
  const payloadBytes = msgpackEncode(payload);
  const hash = Hash.sha256(payloadBytes);
  const signature = privateKey.sign(hash);

  return TransactionBuilder.encode('city_found', {
    ...payload,
    signature: signature.toString('hex'),
  });
}
```

## Steps

1. Add `privateKey` parameter to `buildCityFound` and other payload builders
2. Sign the msgpack-encoded payload (excluding the signature field)
3. Include the hex-encoded signature in the payload
4. On the indexer side, verify the signature matches the `founder_pubkey` before indexing
5. Update the Elixir `build_city_found/1` to accept and verify signatures similarly
