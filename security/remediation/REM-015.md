# REM-015: Unify Type Codes Across Core and Node

**Fixes:** FINDING-015
**Priority:** LOW
**Effort:** Small (< 1 hour)

## Problem

Core uses `"LOCUS"` prefix with territory type codes. Node uses `"locus"` prefix with ghost type codes.

## Recommended Fix

Define a single, canonical type code registry in a shared module or specification document. Use a single prefix.

**Option A:** Merge into a single namespace

```
"LOCUS" prefix for ALL messages:
  0x01-0x0F: City operations
  0x10-0x1F: Territory operations
  0x20-0x2F: Object operations
  0x30-0x3F: Heartbeat/presence
  0x40-0x4F: Ghost operations
  0x50-0x5F: Governance
  0x60-0x6F: Economic (UBI, etc.)
```

**Option B:** Version-prefix to distinguish

Use `"LOCUS"` for all, add a sub-protocol byte:
```
OP_RETURN "LOCUS" {version:1} {sub_protocol:1} {type:1} {payload}
  sub_protocol 0x01 = territory layer
  sub_protocol 0x02 = ghost layer
```

## Steps

1. Choose unified scheme
2. Update `core/lib/locus/transaction.ex` type codes
3. Update `node/lib/locus/tx_builder.ex` type codes and prefix
4. Update `client/src/constants/opcodes.ts` to match
5. Update indexer parser to handle the unified format
