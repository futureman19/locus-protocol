# REM-011: Fix Fibonacci Sequence Exponential Complexity

**Fixes:** FINDING-011
**Priority:** LOW
**Effort:** Small (< 30 minutes)

## Current Code

```elixir
# core/lib/locus/fibonacci.ex:40-44
def sequence(n) when n > 2 do
  sequence(n - 1) ++ [next_fib(List.last(sequence(n - 1)), List.last(sequence(n - 1) |> Enum.drop(-1)))]
end
```

## Recommended Fix

```elixir
@spec sequence(non_neg_integer()) :: list(non_neg_integer())
def sequence(0), do: []
def sequence(1), do: [1]
def sequence(n) when n >= 2 do
  Enum.reduce(2..n-1, [1, 1], fn _i, [b, a | _] = acc ->
    [a + b | acc]
  end)
  |> Enum.reverse()
end
```

This runs in O(n) time and O(n) space.

Note: Since `sequence/1` is not currently used in any production code path (the lookup table in `blocks_for_citizens/1` is used instead), this is low priority. However, it should be fixed to prevent accidental use by future developers.
