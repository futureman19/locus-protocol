# REM-005: Replace Float Arithmetic with Integer Bit-Shifting

**Fixes:** FINDING-005
**Priority:** HIGH
**Effort:** Small (< 1 hour)

## Current Code

```elixir
# core/lib/locus/staking.ex:165-167
def territory_tax(base_cost, territory_number) when territory_number >= 1 do
  trunc(base_cost * :math.pow(2, territory_number - 1))
end
```

## Recommended Fix (Elixir)

```elixir
@doc """
Calculate the cost for claiming the Nth territory.
Progressive tax: base_cost * 2^(N-1)
Uses integer bit-shifting for deterministic results.
"""
@spec territory_tax(non_neg_integer(), pos_integer()) :: non_neg_integer()
def territory_tax(base_cost, territory_number) when territory_number >= 1 do
  # Elixir integers are arbitrary precision — no overflow risk
  base_cost * Bitwise.bsl(1, territory_number - 1)
end
```

`Bitwise.bsl(1, n)` computes `2^n` as a pure integer operation. Elixir's arbitrary-precision integers mean this works correctly for any value of `territory_number`.

## Recommended Fix (TypeScript)

```typescript
export function progressiveTax(baseCost: number, propertyNumber: number): number {
  if (propertyNumber <= 0) throw new Error('propertyNumber must be >= 1');
  if (propertyNumber > 53) throw new Error('propertyNumber too large for safe integer');
  // Use BigInt for the exponentiation, convert back
  const multiplier = Number(1n << BigInt(propertyNumber - 1));
  return baseCost * multiplier;
}
```

Or for full safety:

```typescript
export function progressiveTax(baseCost: bigint, propertyNumber: number): bigint {
  if (propertyNumber <= 0) throw new Error('propertyNumber must be >= 1');
  return baseCost * (1n << BigInt(propertyNumber - 1));
}
```
