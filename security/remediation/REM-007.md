# REM-007: Use Cryptographic RNG for Heartbeat Nonce

**Fixes:** FINDING-007
**Priority:** MEDIUM
**Effort:** Small (< 30 minutes)

## Current Code

```typescript
// client/src/modules/transaction-builder.ts:188
nonce: params.nonce ?? Math.floor(Math.random() * 0xffffffff),
```

## Recommended Fix

```typescript
import { randomBytes } from 'crypto';

static buildHeartbeat(params: HeartbeatParams): Buffer {
  const nonce = params.nonce ?? randomBytes(4).readUInt32LE(0);
  return TransactionBuilder.encode('heartbeat', {
    // ...
    nonce,
  });
}
```

For browser environments:

```typescript
const nonce = params.nonce ?? crypto.getRandomValues(new Uint32Array(1))[0];
```

## Steps

1. Replace `Math.random()` with `crypto.randomBytes(4).readUInt32LE(0)` (Node.js)
2. If browser support is needed, use `crypto.getRandomValues()`
3. Ensure the nonce is 32-bit unsigned integer to match the Elixir implementation
