# REM-009: Complete or Remove Payment Channel Endpoints

**Fixes:** FINDING-009
**Priority:** MEDIUM
**Effort:** Large (1-2 weeks for full implementation) or Small (< 1 hour to remove)

## Options

### Option A: Remove stubbed endpoints (recommended for pre-mainnet)

Remove the `/channels` routes entirely and return 501 Not Implemented:

```rust
pub async fn open_channel(/* ... */) -> Result<...> {
    Err((
        StatusCode::NOT_IMPLEMENTED,
        Json(json!({"error": "Payment channels not yet implemented"})),
    ))
}
```

### Option B: Implement payment channels (mainnet requirement)

Full implementation requires:
1. Funding transaction verification (confirm UTXO exists on-chain)
2. State channel updates with bilateral signatures
3. Dispute resolution with timelock
4. Settlement transaction construction
5. On-chain monitoring for unilateral closes

This is a significant engineering effort and should be tracked as a separate work item.

## Steps (Option A)

1. Mark all three endpoints as returning 501
2. Add a feature flag for payment channels
3. Document that payment channels are not yet available
4. Remove fake response data that could mislead integrators
