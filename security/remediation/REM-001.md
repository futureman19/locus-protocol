# REM-001: Remove Host Environment Inheritance from WASM Sandbox

**Fixes:** FINDING-001
**Priority:** CRITICAL — Must fix before any deployment
**Effort:** Small (< 1 hour)

## Current Code

```rust
// ghost/runtime/src/runtime/engine.rs:62-65
let wasi = WasiCtxBuilder::new()
    .inherit_stdio()
    .inherit_env()
    .build();
```

## Recommended Fix

```rust
let wasi = WasiCtxBuilder::new()
    // DO NOT inherit stdio in production — ghosts should not write to host stdout
    // .inherit_stdio()
    // DO NOT inherit env — this leaks ALL host secrets
    // .inherit_env()
    // Only expose explicitly allowed environment variables:
    .env("GHOST_ID", &ghost_id)
    .env("NETWORK", "mainnet")
    .build();
```

## Steps

1. Remove `.inherit_env()` from `create_store()`
2. Remove `.inherit_stdio()` — route ghost output through the logging host function instead
3. If specific environment variables are needed by ghosts, allowlist them explicitly using `.env(key, value)`
4. Add a test that verifies WASM modules cannot read host environment variables
5. Document which environment variables are available to ghosts in the ghost developer guide

## Verification

```rust
#[test]

    std::env::set_var("SECRET_KEY", "supersecret");
```
