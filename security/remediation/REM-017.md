## Recommended Fix

Add a block height provider that reads from the indexer or a blockchain API:

```rust
use std::sync::atomic::{AtomicU32, Ordering};
use std::sync::Arc;

pub struct BlockHeightProvider {
    current_height: Arc<AtomicU32>,
}

impl BlockHeightProvider {
    pub fn new() -> Self {
        Self {
            current_height: Arc::new(AtomicU32::new(0)),
        }
    }

    pub fn get(&self) -> u32 {
        self.current_height.load(Ordering::Relaxed)
    }

    pub fn update(&self, height: u32) {
        self.current_height.store(height, Ordering::Relaxed);
    }
}
```

Add a background task that polls the blockchain for new blocks:

```rust
async fn block_height_updater(provider: Arc<BlockHeightProvider>, api_url: String) {
    loop {
        if let Ok(height) = fetch_block_height(&api_url).await {
            provider.update(height);
        }
        tokio::time::sleep(Duration::from_secs(30)).await;
    }
}
```

## Steps

1. Create `BlockHeightProvider` with atomic read/write
2. Spawn a background updater task on runtime startup
3. Inject the provider into `HostFunctions`
4. Replace hardcoded `800000` with `self.block_height_provider.get()`
